W04 - Deploying a Web Server

The objectives of this practical are:

  • Allow the server to server more than one connection before exciting
  • Handles runtime errors eloquently such as ERROR 404
  • log file contains: Client IP, file requested, timestamp, allow logfile to be requested
  • Runs as background process
  • Multi threading new thread for each connection
  • Start up and closing functionality with option of port number, log file, document-root
  • initially returns tuple of TCP connection before main request
  • instructions on how to restart
  • GeoLocation report
  • JUnit test HttpResponse
  • Create cookies
  • POST and HEAD
